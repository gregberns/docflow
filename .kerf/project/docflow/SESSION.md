# DocFlow — Session log

The kerf `plan` jig (v1) expects a `SESSION.md` at the work root. This project tracks per-session state in `spec.yaml` (`sessions:` field) and uses a separate `HANDOFF.md` at the repo root for between-session continuity. No standalone session narrative is maintained here.

For session resumption: read `/HANDOFF.md` (repo root). For session history: see `spec.yaml` and `git log`.
